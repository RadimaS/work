Чем полезна

Пиксель в пиксель

Понимаю, как важно сделать верстку точно по макету, поэтому верстаю пиксель в пиксель, если потребуется. В процессе работы накладываю дизайн-макет на верстку, чтобы убедиться, что расхождений нет.

Адаптивная верстка

Верстаю адаптивно. Верстку проверяю при помощи google-расширения для имитации работы сайта на различных устройствах. Поэтому проекты одинаково хорошо выводятся на разных экранах.

Стили на препроцессорах

Стили пишу с помощью препроцессора SASS, поэтому процесс их написания ускоряется, нужный участок кода находится легко и получается делать гибкие настройки.Например, для повторяющегося значения создаю переменную и использую её.

Хостинг, домен, Git

Умею настраивать хостинг, привязывать домен и регистрировать SSL-сертификат. Сам проект обычно храню в репозитории на Github. Правда, в командной разработке еще не участвовала, поэтому знание гита на уровне: создать репозиторий, клонировать его, внести изменения, сохранить коммиты и отправить их в репозиторий.

Информационное веб-приложение "Beymax"

Ссылка

beymax.herokuapp.com

Срок

3 месяца

Задача

Создать веб-приложение для помощи абитуриентам в выборе образования.

Дедлайн

Так как это дипломный проект, дедлайн - представить на защите готовое приложение. Всего на разработку было 5 месяцев, а выполнена работа за 3 месяца.

Описание проекта

Как уже было сказано, этот проект создан в рамках дипломной работы. Тема была предложена мною и одобрена руководством университета.

Это информационное приложение, содержащее программу обучения ведущих вузов Чеченской Республики. В нем представлены следующие данные: описание вуза; его факультеты и институты; по факультету/институту доступны направления; по направлениям отображаются их основные предметы с лекциями.

Дизайн проекта полностью разработан мной. Макет первой страницы был сделан в Wix.com, а последующие страницы разработны без макета. Больше всего времени при разработке клиентской части потрачено на иллюстрации, которые подходят под название факультета/института.

Клиентская часть построена на HTML/CSS + JS (самая малость), а сервер - Node.js + Express. В качестве базы данных использована MongoDB. Больше всего сложностей в разработке серверной части доставило заполнение базы данных: необходимо было создать для каждого выводимого объекта свою коллекцию, заполнить ее и связать с другой коллекцией, подходящей по логике.

Все лекции (общее количество которых почти 400) собраны и отредактированы так же мною самостоятельно. Проделана огромная работа, зато приобретен и закреплен навык работы с большим количеством информации.

Веб-сервис туристического агентства

Ссылка

tour-plan.com

Срок

3 недели

Задача

Создать веб-сервис по подбору и бронированию отелей, перелетов, путёвок.

Дедлайн

Данный проект создан в рамках учебного курса по верстке. Так как проект нелегкий, на него выделено было больше всего времени - приблизительно 3 недели. Работа была выполнена в срок, а впоследствии (по собственному желанию) немного доработана.

Описание проекта

Как уже было сказано, это веб-сервис для подбора и бронирования мест отдыха. Бронирование осуществляется через почту, что означает наличие на сайте php-формы для отправки писем.

Php-форм на сайте всего три: для бронирования, для подписки и для вопросов/сообщений. Их работа проверена перенаправлением писем на собственную почту.

Дизайн проекта создан по макету. Однако макет был предоставлен только для компьютерной версии. Макет для мобильной конструировался самостоятельно.

Проект содержит в себе большую часть важных для верстки элементов: адаптивность, модальные окна, php-формы, слайдер.

Сложнее всего было расположить все мелкие детали сайта по pixelPerfect: поправляешь один объект - смещается другой. Но результат того стоил + это прививает стремление к лучшему результату.
